Media Blast Boosters™ — PR Dashboard Context Brief> Provide this file + PRDashboard-Enhanced.jsx at the start of any new session.---What This App IsMedia Blast Boosters™ is a React-based PR and media intelligence dashboard built for digital marketing/PR use. It was originally generated by Softgen (Vite + React + TypeScript + Tailwind + shadcn/ui + Recharts + Supabase), then heavily enhanced. The entire app lives in one self-contained file: `src/components/PRDashboard.tsx`.---Tech StackFramework: React (JSX, hooks) — no TypeScript types used in practiceStyling: Inline styles throughout (no Tailwind classes except `field-input`, `btn-primary`, `btn-secondary`, `card`, `animate-fadein`, `font-display` which are defined in the global CSS)Charts: Recharts (RadarChart for Competitor Analysis)Icons: Custom inline SVG icon components (no external icon library)Fonts: Outfit (display) + DM Sans (body) via Google FontsStorage: `window.storage` (Softgen's persistent key-value store) — NOT localStorageAI: Anthropic Claude API (`claude-sonnet-4-20250514`) called directly via `fetch`---API ArchitectureTwo core API functions (top of file, outside component):```jscallClaude(content, system, maxTokens, apiKey)  // standard completioncallGemini(prompt, apiKey)                       // Claude + web_search tool```Two wrapper functions (inside component, auto-inject saved key):```jsai(content, system, tokens)   // → callClaude with claudeApiKeyaiW(prompt)                   // → callGemini with claudeApiKey```Claude API Key:Stored in `window.storage` under key `"mbb:claudeKey"`Entered by user in Settings modal (`sk-ant-…`)Required when deployed outside Claude sandbox (Softgen deployment)Auto-injected via `x-api-key` header + `anthropic-version: 2023-06-01`---Persistent Storage Keys (window.storage)Key	Contents`mbb:companyData`	JSON — full company profile object`mbb:claudeKey`	Claude API key string`mbb:geminiKey`	Legacy — no longer used, can be removed---Company Data Object (EMPTY_COMPANY shape)```js{  name: "",               // Company name  industry: "",           // Industry/sector (used by all AI features)  websiteUrl: "",         // Website URL  googleProfileUrl: "",   // Google Business Profile URL  summaryFileUrl: "",     // Link to .txt summary file  quoteAttribution: "",   // "CEO Name — Title, Company"  about: "",              // 2-3 sentence company description  services: "",           // Comma-separated list of services  address: "",            // Full street address  phone: "",  email: ""}```---5 Main Tabs1. Trending TopicsData source: Google News RSS via multi-proxy chain (allorigins → corsproxy → thingproxy)Queries: Built from `industry` + first 2 services (2 batches of 6, deduplicated → 12 max)Pagination: 6 per page, 2 pagesCards show: Title, summary, source, date (MM/DD/YYYY), 🔥 HIGH badgeExternal link: `window.open(t.url)` — opens real articleButtons per card: 💡 Content Ideas (calls `ai()`), ✍️ Create PR (sets selectedTopic → switches to PR tab)Footer: "Rescan Topics" naked button (matches Competitor Analysis style)Empty state: White card with icon, description, Run Analysis button2. Competitor AnalysisTrigger: Manual "Scan Market" buttonAI call: `ai()` — single callClaude, ~8 secondsOutput: RadarChart (5 axes: AI Citation, Media Authority, News Volume, Sentiment, Topic Leadership), 3 competitor cards with gap analysis, 5 Competitive Intelligence bullet pointsFooter: "Rescan Market" naked button3. Trust Assets (minimal changes from original)Widget style selector, verification URL input, resolution options4. PR CreatorFields: Selected topic (from Trending Topics), Company Name, Quote Attribution, Main Focus dropdown, Theme & Style dropdown, Article Length, Keywords (max 2), About text, Key Quote, Video URL, Maps Embed, Featured ImageAI call: `ai()` with large prompt (~2000 tokens) → generates full HTML press releaseRefine with AI: Up to 5 refinements via dialog, calls `ai()`Output view: Rendered HTML preview, Copy HTML button, Download .html button5. Orders (minimal changes from original)Order list with status badges, order detail modal---Company Data ModalTwo modes: 🤖 AI Company Data | ✍️ Manual EntryAI mode — 3 source options (tab switcher):🌐 Website URL → crawls up to 9 priority pages📍 Google Profile URL → extracts from Google Business page📄 Summary File URL → reads .txt fileCrawl engine: `aiW()` (Claude + web_search tool) — fetches pages server-side, no CORS issuesSource rules in prompt: name/industry from Home, quoteAttribution from About/Team, address/phone/email from Contact only, services from Services pageAfter crawl: Switches to Manual mode pre-filled for review before savingLive crawl log: Shows each page with ✓/– statusManual form sections: Company Info, About Company, List of Services, Contact DataSave: Writes to `window.storage`, closes modal---Settings ModalClaude API Key — `sk-ant-…` input, saved to storage, green ✓ badge when setCustom PR Prompt — override the default PR generation prompt with placeholders: `{companyName}`, `{industry}`, `{websiteUrl}`, `{mainFocus}`, `{theme}`, `{targetWords}`, `{keywordsText}`, `{about}`, `{quote}`, `{quoteAttribution}`---Header Bar (sticky, dark #0b1120)Left: Media Blast Boosters™ logo + ZapIconCenter: 5-tab navigationRight: Company Data button (green dot when data saved) + Settings gear icon---Key Design PatternsNo sidebar — removed, full-width layout max 940px centeredIncremental changes preferred — use str_replace for targeted edits, not full rewritesError handling: Errors shown inside white cards only (no global red banner at top)Toast notifications: 3.5s auto-dismiss, top-rightDropdowns: Custom built (no native select for styled ones), click-outside closes via state togglesAll modals: Fixed overlay, centered card, z-index 50---Known Constraints`window.open(url, "_blank")` required for links — `<a href>` clicks are swallowed in Claude sandbox`window.storage` only — localStorage is not supported in Softgen/Claude artifactsExternal fetch calls (RSS proxies) only work outside the Claude sandboxInside Claude sandbox, only `api.anthropic.com` is reachable---DeploymentBuilt for Softgen
